# Quick Start

Get your first app built with Canaveral in under 5 minutes.

## Initialize Your Project

<Steps>
1. Navigate to your project directory:
   ```bash
   cd my-app
   ```

2. Run the init command:
   ```bash
   canaveral init
   ```

   This creates a `canaveral.yaml` (or `canaveral.toml`) configuration file with auto-detected settings. Use `--force` to overwrite an existing config, or `--yes` to skip interactive prompts.

3. Review the generated configuration:
   ```yaml title="canaveral.yaml"
   name: MyApp

   versioning:
     strategy: semver
     tag_format: "v{version}"

   git:
     branch: main
     require_clean: true

   changelog:
     enabled: true
     file: CHANGELOG.md
   ```
</Steps>

<Aside type="tip">
  Canaveral auto-detects your framework based on project files like `pubspec.yaml` (Flutter), `app.json` (Expo), or `package.json` with `react-native`.
</Aside>

## Build Your App

### iOS Build

```bash
# Debug build
canaveral build --platform ios --profile debug

# Release build
canaveral build --platform ios --profile release
```

### Android Build

```bash
# Debug APK
canaveral build --platform android --profile debug

# Release AAB (for Play Store)
canaveral build --platform android --profile release
```

## Run Tests

```bash
# Run tests (auto-detects framework)
canaveral test

# Run tests for a specific platform
canaveral test --platform ios

# With coverage report
canaveral test --coverage

# With a test filter
canaveral test --filter "login"

# With JUnit output for CI
canaveral test --reporter junit --output test-results/results.xml

# Smart test selection (only tests covering changed code)
canaveral test --smart --base main

# Only test affected packages in a monorepo
canaveral test --affected --base main
```

## Version Management

```bash
# Calculate the next version based on commits since last tag
canaveral version

# Specify release type
canaveral version --release-type patch  # 1.0.0 -> 1.0.1
canaveral version --release-type minor  # 1.0.0 -> 1.1.0
canaveral version --release-type major  # 1.0.0 -> 2.0.0

# Show current version
canaveral version --current

# Version a specific package in a monorepo
canaveral version --package my-lib
```

## Distribution

### TestFlight (iOS)

<Steps>
1. Build your iOS app:
   ```bash
   canaveral build --platform ios --profile release
   ```

2. Upload to TestFlight:
   ```bash
   canaveral testflight upload build/ios/App.ipa --changelog "Bug fixes"
   ```

3. Check processing status:
   ```bash
   canaveral testflight status
   ```
</Steps>

### Google Play (Android)

<Steps>
1. Build your Android app:
   ```bash
   canaveral build --platform android --profile release
   ```

2. Upload to internal track:
   ```bash
   canaveral publish google-play --artifact build/android/app.aab --track internal
   ```
</Steps>

## Certificate Management

Set up certificate sync for your team:

<Steps>
1. Initialize certificate storage:
   ```bash
   # Using Git
   canaveral match init --storage git --git-url git@github.com:org/certs.git --team-id TEAMID

   # Using S3
   canaveral match init --storage s3 --bucket my-certs-bucket --team-id TEAMID
   ```

2. Sync certificates (creates if needed):
   ```bash
   canaveral match sync --profile-type development
   canaveral match sync --profile-type appstore
   ```

3. In CI, use read-only mode:
   ```bash
   canaveral match sync --profile-type appstore --readonly
   ```
</Steps>

## CI/CD Integration

### GitHub Actions

Create `.github/workflows/build.yml`:

```yaml
name: Build
on: [push]

jobs:
  build-ios:
    runs-on: macos-14
    steps:
      - uses: actions/checkout@v4

      - name: Install Canaveral
        run: |
          curl -fsSL https://get.canaveral.dev | sh
          echo "$HOME/.canaveral/bin" >> $GITHUB_PATH

      - name: Sync Certificates
        run: canaveral match sync --profile-type appstore --readonly
        env:
          MATCH_GIT_URL: ${{ vars.MATCH_GIT_URL }}
          MATCH_PASSWORD: ${{ secrets.MATCH_PASSWORD }}

      - name: Build iOS
        run: canaveral build --platform ios --profile release

      - name: Upload to TestFlight
        run: canaveral testflight upload build/ios/*.ipa
        env:
          APP_STORE_CONNECT_API_KEY_ID: ${{ secrets.APP_STORE_CONNECT_API_KEY_ID }}
          APP_STORE_CONNECT_ISSUER_ID: ${{ secrets.APP_STORE_CONNECT_ISSUER_ID }}
          APP_STORE_CONNECT_API_KEY_PATH: ${{ secrets.APP_STORE_CONNECT_API_KEY_PATH }}
```

<Aside type="note">
  Use the global `--format json` flag for machine-readable output in CI pipelines.
</Aside>

## Troubleshooting

Run the doctor command to diagnose issues:

```bash
canaveral doctor
```

Common fixes:
- **Missing Xcode**: `xcode-select --install`
- **Missing Android SDK**: Set `ANDROID_HOME` environment variable
- **Missing Flutter**: Install from https://flutter.dev
- **Certificate issues**: Run `canaveral match sync --force`
