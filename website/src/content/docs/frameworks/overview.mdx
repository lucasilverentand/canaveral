# Framework Overview

Canaveral provides framework-agnostic build, test, and release automation. Each framework adapter auto-detects your project type and provides optimized workflows.

## Auto-Detection

Canaveral automatically detects your framework based on project markers:

| Framework | Detection Method | Confidence |
|-----------|-----------------|------------|
| Flutter | `pubspec.yaml` with `flutter:` | 95% |
| Expo | `app.json` with `expo` field | 95% |
| React Native | `package.json` with `react-native` | 90% |
| Native iOS | `.xcworkspace` or `.xcodeproj` | 95% |
| Native Android | `build.gradle` with `android` plugin | 95% |
| Tauri | `src-tauri/tauri.conf.json` | 95% |

To manually specify a framework:

```bash
canaveral build --framework flutter --platform ios
```

Or specify the framework in the build command:

```bash
canaveral build --framework flutter --platform ios
```

## Common Commands

All frameworks share the same command interface:

```bash
# Build
canaveral build --platform <platform> --profile <profile>

# Test
canaveral test --platform <platform>

# Version
canaveral version --release-type <patch|minor|major>

# Distribute
canaveral testflight upload <artifact>
canaveral publish google-play --artifact <artifact> --track internal
```

<Aside type="tip">
  The same commands work regardless of framework. Canaveral handles the framework-specific details automatically.
</Aside>

## Framework-Specific Features

Each framework has unique capabilities:

- **Flutter**: Widget tests, flavor-based builds, Dart analyzer
- **Expo**: EAS Build integration, OTA updates via EAS Update
- **React Native**: Metro bundler, CodePush OTA updates
- **Native iOS**: Multiple schemes, CocoaPods/SPM integration
- **Native Android**: Gradle flavors, ProGuard/R8 configuration
- **Tauri**: Multi-platform bundles, auto-updater

See the individual framework guides for detailed documentation.
