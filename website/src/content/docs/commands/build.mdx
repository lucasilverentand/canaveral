# canaveral build

Build your app for iOS, Android, or desktop platforms. Canaveral auto-detects your framework and uses the appropriate build tools.

## Synopsis

```bash
canaveral build --platform <PLATFORM> [OPTIONS] [-- <EXTRA_ARGS>...]
```

## Options

| Option | Short | Description | Default |
|--------|-------|-------------|---------|
| `--platform <PLATFORM>` | `-p` | Target platform: `ios`, `android`, `macos`, `windows`, `linux`, `web` | **Required** |
| `--profile <PROFILE>` | | Build profile: `debug`, `release`, `profile` | `release` |
| `--flavor <FLAVOR>` | | Build flavor (e.g., `production`, `staging`) | - |
| `--framework <FRAMEWORK>` | | Override framework detection: `flutter`, `expo`, `react-native`, `tauri`, `native-ios`, `native-android` | Auto-detected |
| `--output <DIR>` | `-o` | Output directory for artifacts | `build/` |
| `--version <VERSION>` | | Override version string | From config |
| `--build-number <NUM>` | | Override build number | Auto |
| `--dry-run` | | Show what would be built without building | `false` |
| `--skip-checks` | | Skip pre-build checks | `false` |
| `--signing-identity <ID>` | | Code signing identity (iOS/macOS) | From config |
| `--provisioning-profile <PROFILE>` | | Provisioning profile name (iOS) | From config |
| `--team-id <ID>` | | Apple Developer Team ID | From config |
| `--keystore <PATH>` | | Android keystore path | From config |
| `--key-alias <ALIAS>` | | Android keystore key alias | From config |
| `-- <EXTRA_ARGS>` | | Additional arguments passed to the underlying build tool | - |

### Global Options

| Option | Description |
|--------|-------------|
| `--format json` | Output build results as JSON |
| `--verbose` | Show detailed build output |
| `--quiet` | Suppress output except errors |

## Examples

### Basic Builds

```bash
# Build for iOS (release)
canaveral build --platform ios

# Build for Android (release)
canaveral build --platform android

# Debug build
canaveral build --platform ios --profile debug
```

### With Flavor and Signing

```bash
# Specify flavor (Android)
canaveral build --platform android --flavor production

# Specify signing identity (iOS)
canaveral build --platform ios --signing-identity "Apple Distribution: My Company"

# Specify keystore (Android)
canaveral build --platform android --keystore release.keystore --key-alias mykey
```

### Pass Extra Arguments to Build Tool

```bash
# Pass extra args to the underlying framework's build command
canaveral build --platform ios -- -allowProvisioningUpdates
canaveral build --platform android -- --stacktrace
```

### Dry Run

```bash
# Show what would be built
canaveral build --platform ios --dry-run
```

## Output

### Text Format (Default)

```
Building MyApp for iOS (release)...
  Framework: Flutter
  Configuration: Release

Build complete!
  Artifact: build/ios/MyApp.ipa
  Size: 45.2 MB
  Duration: 2m 34s
```

```bash
canaveral build --platform ios --format json
```

```json
  "success": true,
  "platform": "ios",
  "profile": "release",
  "framework": "flutter",

      "type": "ipa",
      "size": 47420416
    }
  ],
  "duration_ms": 154000
}
```
